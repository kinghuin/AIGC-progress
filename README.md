
# AIGC进展

已有的以Awesome为代表的集锦仓库先分类再找仓库，这种方式不能刻画出仓库的全部能力，也不方便跟踪AIGC领域快速涌现的新仓库。本仓库每天将以仓库为第一维度，通过打Tag的方式对它们进行更加全面的刻画，您可以通过时间线看到最新进展，也可以通过Tag方便地找到AIGC相关的新模型、新数据集、新框架、新应用等。

Existing curated repositories, represented by Awesome, typically classify repositories first and then locate them. This approach fails to capture the full capabilities of the repositories and makes it inconvenient to track the rapid emergence of new repositories in the AIGC field. In this repository, we prioritize repositories as the primary dimension and provide a more comprehensive description of them through tagging. You can follow the timeline to see the latest developments and easily find new models, datasets, frameworks, applications, and more related to AIGC through the tags.

![image](./img/repository_month_count.png)


## Tag介绍（Tag Introduction）:

* [文本（Txt）](tags/文本.md)、[图像（Img）](tags/图像.md)、[语音（Voice）](tags/语.md)、[视频（Video）](tags/视频.md): 输入或输出存在文本、图像、语音、视频（Input or Output is Txt/Img/Voice/Video）。

* [数据（Data）](tags/数据.md)、[通用模型（General Model）](tags/通用模型.md)、[定制模型（Custom Model）](tags/定制模型.md)：提供了相关资源，定制模型包含法律、医学等垂类模型以及代码生成、SQL生成等模型（Related resources are provided. Custom model include domain-specific models such as legal and medical models, as well as models for code generation and SQL generation.）

* [预训练（Pre-training）](tags/预训练.md)、[训练（training）](tags/训练.md)、[强化（reinforcement, like RLHF）](tags/强化.md)、[压缩（compression）](tags/压缩.md)、[推理（inference）](tags/推理.md)：提供了相关代码 （Related code is provided.）

* [交互（Interaction）](tags/交互.md)：提供更好的界面或者API去使用模型。（Provides a better interface or API to use the model.）

* [插件（Plugin）](tags/插件.md)：增强模型的某个功能。（Enhances a specific functionality of the model.）

* [工具（Tool）](tags/工具.md): 使用模型去完成某件任务。（Utilizes the model to accomplish some task.）

* [文档（Document）](tags/文档.md):教程、集锦等。（Tutorials, compilations, and more.）

* [智能体（Agent）](tags/智能体.md)

* [可商用 (Commercial Use)](tags/可商用.md)

* [中文（Chinese）](tags/中文.md)：中文模型或中文数据集。（Chinese Model or Chinese Dataset）

* [其它 (others)](tags/其它.md)
